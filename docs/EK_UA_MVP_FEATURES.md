# MVP-функции на основе аудита EK.UA

Дата: 23 февраля 2026  
Источник: `docs/EK_UA_FUNCTIONALITY_FULL_AUDIT.md`

## 1) Цель MVP

Запустить рабочий сценарий выбора и покупки товара:
1. пользователь находит товар через каталог/поиск;
2. быстро отфильтровывает варианты;
3. сравнивает цены по магазинам;
4. переходит в магазин на покупку;
5. может сохранить товар и вернуться позже.

## 2) Критично для MVP (Must Have)

### 2.1 Глобальная навигация
- Глобальный поиск по каталогу.
- Навигация по категориям.
- Базовая авторизация (login/register/logout).
- Публичные страницы: главная, каталог, карточка товара.

### 2.2 Каталог (основной экран конверсии)
- Листинг товаров категории.
- Фильтры:
  - поиск по тексту;
  - диапазон цены;
  - бренд;
  - магазин/продавец;
  - динамические атрибуты категории.
- Сортировка: popular, relevance, price asc/desc, newest.
- Пагинация страниц (вместо `load more`).
- Фиксированный блок фильтров на desktop при скролле ленты.
- Сброс фильтров и отображение количества активных фильтров.

### 2.3 Карточка товара (PDP)
- Основная информация о товаре: название, бренд, категория, фото.
- Блок предложений магазинов с ценами.
- Сортировка офферов (цена/доставка/условный рейтинг продавца).
- Актуальность офферов (время обновления, in stock/out of stock).
- Явный CTA перехода в магазин (`Buy`/внешняя ссылка).
- Таблица характеристик (specs).

### 2.4 Пользовательские функции удержания
- Избранное (добавить/удалить, отдельная страница).
- Недавно просмотренные (история просмотров, отдельная страница).
- Профиль пользователя:
  - просмотр и редактирование базовых полей;
  - сохранение профиля через API;
  - локальный backup черновика.

### 2.5 Базовые доверительные элементы
- Сообщения состояний: loading/empty/error в каталоге и карточке товара.
- Понятные метки цены и наличия.
- Базовый SEO для PDP (structured data Product/Offer).

## 3) Важно, но после MVP (Phase 2)

- Полноценное сравнение товаров (comparison matrix).
- Отзывы, Q&A, обсуждения/форум.
- Рейтинги категории, популярные запросы, энциклопедические разделы.
- Контентные разделы (статьи, редакционные подборки).
- Расширенные уведомления (price drop/stock) с серверной синхронизацией.
- Продвинутая безопасность профиля (смена пароля, 2FA, управление сессиями).
- B2B-кабинет магазинов и рекламные инструменты.

## 4) Не включать в MVP (пока не приоритет)

- Мультистрана/мультиязык как отдельный проектный трек.
- Dark mode как обязательное требование релиза.
- Глубокая аналитика/диаграммы по категориям.
- Сложные community-механики модерации.

## 5) Минимальный критерий готовности MVP

- Пользователь может за <= 2-3 минуты пройти путь:
  `Каталог -> Фильтры -> Карточка -> Сравнение цен по магазинам -> Переход в магазин`.
- Основные страницы стабильно работают на mobile + desktop.
- Нет блокирующих ошибок в flow каталог/карточка/избранное/профиль.
